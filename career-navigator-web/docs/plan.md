## 17. 챕짖챔짤짝챔짧짧챈쨀 Interview Narrative

### 채쨍책짜챔짤짹채쨩챌쨈쨔

> "I built an AI-powered career navigation platform that parses resumes, matches them against job descriptions, and tracks applications. I used it myself during my own job search 창 so I was the first real user."

### 챈 쨍책쩔챈챔징챈짹쨘챌짯챔짧짧챈쨀

**챌쨘채쨩챕쨘쩌 MVP 챌짱책챌 챕챕쨘쩌책째챦쩌**

> "I originally scoped this with way too many features at once. I cut it down to the smallest loop that proves value 창 upload resume, get analysis, match against a job description, track the application 창 and pushed everything else to later phases."

**챕책째챌쨘채쨩챕쨘쩌챌짢챌짯챌쨈챔채쨍챈짱챌쩐책챈짱챦쩌**

> "A number like 82% looks rigorous but isn't meaningful 창 there's no real difference between 82 and 79. I used five interpretable levels backed by a required-skill hit-rate rule. It's more honest and more useful: a job seeker wants to know 'should I apply', and 'good' answers that better than a fake percentage."

**Skill Tree 챌쨘채쨩챕쨘쩌채쨍챈짱책쨀챈 AI 챌챈챦쩌**

> "Fully dynamic generation looked impressive on paper but was unreliable for a live demo. I built curated skill trees for common roles and let the AI handle classification instead of generation 창 same lesson I learned on Lumora, where I replaced raw AI output with a stable mapping layer."

**챌쨘채쨩챕쨘쩌책짹짜챈짯쨌책짧책챈채쨍챈짭징챦쩌**

> "Users compare one resume against many jobs. Re-running the LLM analysis every time would waste calls and time, so I analyze once on upload, store the structured result, and reuse it for every job match."

**챌쨘채쨩챕쨘쩌책짹짜챈짯쨌챔짝챕쨌챈채쩔책짯챦쩌챔쨌채쩍 책짝채쨍책책째챈징챌찾챌짠챌챌짢책짰책째짹책짧찾챕챔쩌짱채쨍채쨍챈짢짙챦쩌**

> "Different asset model. In Lumora the photo was just a trigger 창 no lasting value after recognition. Here the resume _is_ the core asset users come back to, so it has to persist. That also meant handling PII and encryption, which Lumora didn't need."
